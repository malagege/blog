超級終端機 指令補全

終端機用久了之後，補全功能其實會直接影響操作效率。尤其在 zsh 環境裡，如果 TAB 補全沒有正常工作，輸入長指令、路徑或參數時就會明顯卡很多。這篇主要記錄我當時在 zsh 與 oh-my-zsh 環境下補全設定的整理。

為什麼指令補全很重要

補全不只是少打一點字而已，實際上還能幫你：

  1. 降低打錯檔名或路徑的機率。
  2. 快速探索某個命令有哪些參數。
  3. 在大量 CLI 工作裡節省時間。

如果 Shell 是主要工作環境，這類基礎功能其實很值得先整理好。

zsh 補全的基本概念

zsh 本身就有不錯的 completion 能力，oh-my-zsh 也帶了不少相關設定。不過在某些環境下，預設不一定會完全符合需求，所以還是可能需要手動補設定檔。

常見做法

當時的做法是把 completion 設定載入到 ~/.zshrc：

1

source ~/.zplug/completion.zsh

加完之後再重新載入設定：

1

source ~/.zshrc

這樣通常就能讓 TAB 補全行為恢復正常或更完整。

如果你是用 zplug

若你的 zsh 套件是用 zplug 管理，相關補全腳本和插件通常會一起放進 zplug 的目錄。這時候重點不是硬背某一個檔案路徑，而是先確認：

  1. 套件是不是有真的安裝成功。
  2. ~/.zshrc 有沒有正確 source 該腳本。
  3. 是否有其他 plugin 順序衝突。

我會怎麼驗證

設定完後，可以直接測幾種場景：

  1. 指令名稱補全。
  2. 檔案路徑補全。
  3. Git 子命令或常用工具參數補全。

如果某些能補、某些不能補，通常就代表不是 Shell 壞掉，而是特定 plugin 或 completion 腳本沒載好。

小結

zsh 的補全功能本來就很強，但前提是相關設定有被正確載入。如果你已經在用 oh-my-zsh 或 zplug，卻發現 TAB 行為不如預期，先回頭檢查 ~/.zshrc 和 completion 腳本來源，通常就能很快縮小問題範圍。
