Sysprep 和 DISM

最近看到新筆電的重製流程又提到 DISM，順手回想起以前做大量部署或系統封裝時常聽到的 Sysprep。這兩個工具常常一起被提到，但其實用途不太一樣。這篇先整理成我自己比較容易記住的方式。

先講結論

如果只想快速記一句話，我會這樣分：

  1. Sysprep 比較像是把 Windows 做系統準備，讓映像可以換機部署。
  2. DISM 比較像是管理、修復、掛載、匯出 Windows 映像。

兩個可以一起出現，但不是互相取代的關係。

Sysprep 是做什麼的

Sysprep 全名是 System Preparation Tool，重點是把目前這台 Windows 安裝好的系統「泛化」。簡單說，就是把跟這台機器太綁定的資訊清掉，讓你之後可以把映像部署到別台機器。

常見用途：

  1. 大量佈署公司電腦。
  2. 建立乾淨母片。
  3. 交機前重設 OOBE 流程。

但 Sysprep 不是拿來當一般備份工具，它偏向部署前整理環境。

DISM 是做什麼的

DISM 是 Deployment Image Servicing and Management。它能做的事情比較廣，像是：

  1. 修復 Windows 映像。
  2. 掛載或卸載 WIM 檔。
  3. 把套件、驅動、功能塞進映像。
  4. 匯出或清理映像內容。

所以如果你看到有人拿 DISM 做系統修復，例如：

1

DISM /Online /Cleanup-Image /RestoreHealth

這是在修目前系統映像，不是在做 Sysprep 那種部署泛化。

什麼情境用哪個

情境 1：你要做母片給很多台電腦用

這時候通常會先整理好系統、安裝必要軟體，最後用 Sysprep 做泛化，再搭配映像備份工具處理。

情境 2：你要修 Windows 元件損毀

這比較偏 DISM 的工作，常會搭配 sfc /scannow 一起用。

情境 3：你要離線修改映像檔

例如要掛載 WIM、加驅動、調整元件，這就是 DISM 的主場。

要注意的坑

Sysprep 不是無限制亂跑。有些版本或某些已安裝 App 狀態下，Sysprep 可能會失敗，像是常見的 Appx package、重置次數、已綁定使用者狀態等問題。

而 DISM 雖然很強，但下錯參數也可能改壞映像，所以正式操作前最好：

  1. 先確認你是在 /Online 還是離線映像。
  2. 知道目前操作的是哪個 WIM 索引。
  3. 先備份再做實驗。

我的理解方式

如果把它們放在同一條流程裡，我會這樣記：

  1. 用 Sysprep 把系統整理成可部署狀態。
  2. 用 DISM 或其他工具去服務映像、修復映像、掛載映像。

這樣就比較不會混在一起。
