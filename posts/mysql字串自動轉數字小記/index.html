mysql字串自動轉數字小記

malagege included in MySQL
 2019-01-17  92 words   One minute 
Contents

最近用 MySQL 做時間比較
想做datetime欄位(0000-00-00 00:00:00) > ''
結果得出來結果是0

之前知道 SQL 沒有 boolean
不過在做時間比較
發現很奇怪
怎麼跟空字串比較結果為0

Google 一下
發現原來 MySQL 會做自動轉型

0 = ''為1
原來空字串會自動轉成0
所以才會得出這個結果
這個不常做，感覺很容易忘記

2019-06-10

最近發現 null 時候，做 where 條件(大於、小於)不會抓到
但是 0000-00-00 可以
所以這是為什麼大家都會這樣用?!
但這樣做 add_date 時候就會有問題
有優點也有缺點
