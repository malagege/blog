解決 Windows 排程錯誤：操作員或系統管理員已拒絕此要求

malagege included in Windows
 2024-05-23  41 words   One minute 
Contents
  • 解決方法
  • 參考文章

最近我在筆電上設定的排程任務都無法正常運行。我發現當筆電蓋上進入休眠模式後，原先設定的排程任務就會執行失敗。更糟糕的是，這種失敗會導致後續的排程任務也無法正常運行，系統會顯示操作員或系統管理員已拒絕此要求的錯誤訊息。

解決方法

在排程任務的設定中，只需勾選再錯過排定的啟動後盡快執行工作選項，即可解決這個問題。黑暗大大提到這裡可能存在時區的 Bug，但我目前並未遇到，所以我先不考慮這個問題。

這邊新增排程預設不會設定，這邊好奇看 Chrome 更新排程，他們確實都有啟用這個設定，所以新設定排程可能需要注意這一項設定。

在新增排程任務時，預設並不會啟用再錯過排定的啟動後盡快執行工作的設定。然而，我發現其他排程(如:Chrome 的更新排程) 確實有啟用這個設定。因此，當我們設定新的排程任務時，可能需要特別注意這一項設定。

參考文章

這兩篇文章都有提到了一種解決方案，設定操作員或系統管理員已拒絕此要求選項，我們可以採用這種方法來解決問題。
